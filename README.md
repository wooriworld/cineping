> **프로젝트명: cineping  
> **버전:** v1.0  
> **작성일:\*\* 2026.03.13

---

## 1. 프로젝트 개요

### 1.1 서비스 소개

영화 상영 스케줄을 자동 수집하여 변경사항이 생겼을 때 알림을 발송하는 서비스. 웹에서는 통합 스케줄을 한눈에 조회할 수 있다.

### 1.2 핵심 문제 정의

국내 멀티플렉스(CGV·롯데시네마·메가박스)는 상영 스케줄 업데이트 시점을 별도로 공지하지 않는다. 사용자는 원하는 영화의 스케줄이 언제 열리는지 알 수 없어, 예매 기회를 놓치지 않으려면 각 사이트를 반복적으로 직접 방문해 확인하는 수밖에 없다.

이는 **정보 비대칭으로 인한 사용자 경험 저하**이며, 특히 인기 영화일수록 스케줄 오픈 직후 좌석이 빠르게 소진되기 때문에 확인 타이밍이 예매 성공 여부를 직접적으로 좌우한다.

본 서비스는 이 반복적인 수동 확인 행위를 자동화하여, **스케줄 변경이 발생한 시점에 사용자에게 즉시 알림을 전달**함으로써 예매 골든타임을 놓치지 않도록 한다.

### 1.3 목표 사용자

- 개봉 예정 영화의 스케줄이 언제 풀리는지 기다리는 영화 팬
- 매일 영화사이트를 직접 들어가 확인하는 수고를 줄이고 싶은 관객

## 2. 주요 기능 정의

### 2.1 스케줄 수집 배치 (GitHub Actions)

| 항목      | 내용                                      |
| --------- | ----------------------------------------- |
| 수집 주기 | 매일 00시, 12시 2회 (GitHub Actions cron) |
| 수집 대상 | CGV, 롯데시네마, 메가박스                 |
| 수집 방식 | Playwright 기반 스크래핑                  |
| 저장소    | Firebase Firestore                        |

**수집 데이터 항목**

- 영화명, 영화관 체인, 지점명
- 상영 날짜, 시작 시간, 종료 시간
- 상영관 타입 (IMAX, 4DX, 일반 등)
- 잔여 좌석 수 / 예매 가능 여부
- 마지막 수집 시각

### 2.2 알림 시스템 (텔레그램)

**알림 트리거 조건 (스크래핑 후 직전 스냅샷과 diff 비교)**

- 새로운 상영 회차가 추가된 경우
- 기존 상영 회차가 삭제된 경우
- 기존 상영 시간·상영관 타입 등이 수정된 경우

**알림 메시지 형식**

```
🎬 [영화명] 스케줄 업데이트
📍 CGV 강남 | IMAX
📅 2026.03.20 (금) 19:30
💺 잔여 좌석: 48석
🔗 예매 바로가기: [링크]
```

### 2.3 웹 서비스 (Vue.js)

#### 페이지 구성

| 페이지 | 경로        | 설명                                  |
| ------ | ----------- | ------------------------------------- |
| 홈     | `/`         | 영화 선택 → 날짜 → 극장 → 스케줄 조회 |
| 설정   | `/settings` | 텔레그램 연동 및 알림 조건 설정       |

#### 주요 UI 컴포넌트

**① 영화 선택 컴포넌트**

- 현재상영중인 영화 목록을 포스터 가로 스크롤로 표시
- 선택된 영화: 제목 굵게 표시
- 포스터 클릭으로 선택 전환, 선택 시 하위 컴포넌트(날짜·극장·스케줄) 자동 갱신

**② 날짜 선택 컴포넌트**

- 오늘부터 7일치 날짜 탭 가로 나열
- 오늘: 강조 + `오늘` 텍스트
- 내일: 텍스트 강조
- 이후: 날짜(숫자) + 요일 기본 스타일
- 해당 날짜에 스케줄 없으면 비활성(disabled) 처리

**③ 필터 컴포넌트**

- 지역 탭: 한국의 각 지역 목록 드롭다운 형태로 중복 선택 가능하게
- 극장 탭: 한국의 영화관 목록 드롭다운 형태로 "전체"가 디폴트
- 선택 시 하단 스케줄 컴포넌트 실시간 필터링

**④ 스케줄 컴포넌트**

- 극장-관별 카드 단위로 시간표 표시
- 극장 별로 시간 버튼이 가로 스크롤로 표시
- 시간 버튼 클릭 시 해당 영화관 예매 페이지로 이동

**설정 페이지 (/settings)**

- 텔레그램 Chat ID 입력 및 연동 테스트 버튼
- 감시 영화 추가/삭제 관리
- 영화별 알림 조건 설정 (신규 회차 추가 / 삭제 / 수정)
- 선호 극장 체인 · 지역 · 상영 타입 필터 지정

## 5. 데이터 모델 (Firestore)

### `movies` 컬렉션

```json
{
  "id": "movie_001",
  "title": "왕과 사는 남자",
  "naverMovieId": "136873",
  "poster": "https://...",
  "rating": "12",
  "runtime": 109,
  "createdAt": "2026-03-14T00:00:00Z"
}
```

> 필드 설명
>
> - rating: 등급 뱃지 값 ("전체" / "12" / "15" / "청불")
> - runtime: 러닝타임(분) — 종료 시간 계산 및 영화 헤더 표시용

---

### `schedules` 컬렉션

```json
{
  "id": "schedule_001",
  "movieId": "movie_001",
  "chain": "메가박스",
  "theater": "메가박스 강동",
  "region": "서울",
  "hall": "1관",
  "date": "2026-03-14",
  "startTime": "16:00",
  "endTime": "18:07",
  "screenType": "2D",
  "totalSeats": 249,
  "seatStatus": "available",
  "bookingUrl": "https://...",
  "lastUpdatedAt": "2026-03-14T12:00:00Z"
}
```

> 필드 설명
>
> - hall: 관 번호 — 동일 극장 내 복수 상영관 구분용
> - endTime: 종료 시간 — 시간 버튼 2번째 줄 표시용
> - totalSeats: 총 좌석수 — "잔여석 | 총석" 표시 및 비율 계산용
> - seatStatus: "available" / "normal" / "almost_full" / "sold_out" — 버튼 색상 분기용
> - region: 지역 필터링용

---

### `users` 컬렉션

```json
{
  "uid": "firebase_uid",
  "telegramChatId": "123456789",
  "favoriteTheaters": ["메가박스 강동", "CGV 강남"],
  "watchlist": [
    {
      "movieId": "movie_001",
      "alertConditions": {
        "newSchedule": true,
        "deletedSchedule": false,
        "modifiedSchedule": true,
        "preferredChains": ["CGV", "메가박스"],
        "preferredRegions": ["서울", "경기"],
        "preferredScreenTypes": ["IMAX", "4DX"]
      }
    }
  ]
}
```

> 필드 설명
>
> - favoriteTheaters: 극장 헤더 ☆ 즐겨찾기 기능 — 즐겨찾기 극장 우선 노출용
> - watchlist: 단순 ID 배열 대신 영화별 세분화 알림 조건 포함 객체 배열
> - preferredRegions / preferredScreenTypes: 지역·상영타입 조건 알림 지원

## 6. 배치 파이프라인 흐름

```
1. GitHub Actions cron 트리거 (매일 00시)
        │
2. 구독 중인 영화 목록 Firestore에서 조회
        │
3. 영화별 스크래핑 실행 (CGV / 롯데 / 메가박스 병렬)
        │
4. 전날 스냅샷과 오늘 수집 결과 diff 비교
   (추가 / 삭제 / 수정 감지)
        │
    ┌───┴───┐
 변경 있음   변경 없음
    │           │
5. Firestore   종료
   스냅샷 갱신
    │
6. 해당 영화 구독 유저 조회
    │
7. Telegram Bot으로 변경 내역 메시지 발송
```

## 8. 알려진 제한 사항

### 8.1 naverMovieId 미수집 영화

네이버 영화 목록 API 응답에서 `naverMovieId`(`code=`)는 카드 내 **예고편 버튼**(`mediaView.nhn?code=XXX`)에서만 추출된다.

예고편이 등록되지 않은 영화는 카드에 해당 링크가 없어 `naverMovieId`를 가져올 수 없다. 이 경우 영화 정보는 저장되지만 스케줄 수집 대상에서 제외된다.

**현재 미수집 영화 예시 (2026-03-18 기준)**

| 영화 | `os=` (검색 파라미터) | `code=` |
|------|----------------------|---------|
| 좀비 랜드 사가: 유메긴가 파라다이스 | 38087004 | 미제공 |
| 레이의 겨울방학 | 40330433 | 미제공 |
| 페루지노. 영원한 르네상스 | 40685420 | 미제공 |
| 남쪽 | 1774641 | 미제공 |
| 결혼해줄래? | 40427339 | 미제공 |

> `os=`는 네이버 내부 검색 파라미터로 스케줄 API의 `code=`와 다른 값이므로 대체 불가.

## 9. 리스크 및 대응 방안

| 리스크                                    | 대응 방안                                       |
| ----------------------------------------- | ----------------------------------------------- |
| 영화관 사이트 구조 변경으로 스크래핑 중단 | 사이트별 파서 모듈 분리, 에러 알림 별도 발송    |

| Telegram Bot API 제한                     | 메시지 큐 처리, 일괄 발송 지양                  |
| Firestore 읽기/쓰기 비용                  | 캐싱 레이어 적용, 불필요한 리얼타임 리스너 제거 |
| 스크래핑 차단 (IP 밴)                     | GitHub Actions IP 풀 활용, User-Agent 로테이션  |
