## 1. 프로젝트 개요

### 1.1 서비스 소개

영화 상영 스케줄을 자동 수집하여 변경사항이 생겼을 때 알림을 발송하는 서비스. 웹에서는 통합 스케줄을 한눈에 조회할 수 있다.

### 1.2 핵심 문제 정의

국내 멀티플렉스(CGV·롯데시네마·메가박스)는 상영 스케줄 업데이트 시점을 별도로 공지하지 않는다. 사용자는 원하는 영화의 스케줄이 언제 열리는지 알 수 없어, 예매 기회를 놓치지 않으려면 각 사이트를 반복적으로 직접 방문해 확인하는 수밖에 없다.

이는 **정보 비대칭으로 인한 사용자 경험 저하**이며, 특히 인기 영화일수록 스케줄 오픈 직후 좌석이 빠르게 소진되기 때문에 확인 타이밍이 예매 성공 여부를 직접적으로 좌우한다.

본 서비스는 이 반복적인 수동 확인 행위를 자동화하여, **스케줄 변경이 발생한 시점에 사용자에게 즉시 알림을 전달**함으로써 예매 골든타임을 놓치지 않도록 한다.

### 1.3 목표 사용자

- 개봉 예정 영화의 스케줄이 언제 풀리는지 기다리는 영화 팬
- 매일 영화사이트를 직접 들어가 확인하는 수고를 줄이고 싶은 관객

## 2. 주요 기능 정의

### 2.1 스케줄 수집 배치 (GitHub Actions cron)

| 항목      | 내용                      |
| --------- | ------------------------- |
| 수집 주기 | 매일 00시, 12시 2회       |
| 수집 대상 | CGV, 롯데시네마, 메가박스 |
| 수집 방식 | Playwright 기반 스크래핑  |
| 저장소    | Firebase Firestore        |

**수집 데이터 항목**

- 영화명, 영화관 체인, 지점명
- 상영 날짜, 시작 시간, 종료 시간
- 상영관 타입 (IMAX, 4DX, 일반 등)
- 잔여 좌석 수 / 예매 가능 여부
- 마지막 수집 시각

### 2.2 알림 시스템 (텔레그램)

**알림 트리거 조건 (스크래핑 후 직전 스냅샷과 diff 비교)**

- 새로운 상영 회차가 추가된 경우
- 기존 상영 회차가 삭제된 경우
- 기존 상영 시간·상영관 타입 등이 수정된 경우

**알림 메시지 형식**

```
🎬 [영화명] 스케줄 업데이트
📍 CGV 강남 | IMAX
📅 2026.03.20 (금) 19:30
💺 잔여 좌석: 48석
🔗 예매 바로가기: [링크]
```

### 2.3 웹 서비스 (Vue.js)

### 페이지 구성

| 페이지       | 경로         | 설명                            |
| ------------ | ------------ | ------------------------------- |
| 홈           | `/`          | 인기 영화 목록 + 최근 업데이트  |
| 영화 검색    | `/search`    | 영화 검색 및 등록               |
| 스케줄 상세  | `/movie/:id` | 영화별 전체 상영 스케줄         |
| 내 알림 설정 | `/settings`  | 텔레그램 연동 및 알림 조건 설정 |

### 주요 UI 컴포넌트

- **영화 카드**: 포스터, 제목, 감시 중 여부 표시
- **스케줄 테이블**: 날짜 × 영화관 × 시간대 그리드
- **필터 바**: 지역, 영화관 체인, 상영 입, 날짜 필터
- **알림 설정 패널**: 텔레그램 Chat ID 연동, 알림 조건 설정

## 4. 기술 스택

### 4.1 프론트엔드

| 항목        | 기술                               |
| ----------- | ---------------------------------- |
| 프레임워크  | Vue 3 (Composition API)            |

| 항목         | 기술                          |
| ------------ | ----------------------------- |
| 런타임       | Node.js                       |

| 스케줄러     | GitHub Actions (cron)         |
| 데이터베이스 | Firebase Firestore            |
| 알림         | Telegram Bot API              |

| 사이트     | 방식                      | 비고          |
| ---------- | ------------------------- | ------------- |

| 롯데시네마 | Playwright + API 인터셉트 | XHR 응답 캡처 |
| 메가박스   | HTTP 직접 요청 (JSON API) | 공개 API 존재 |

## 6. 배치 파이프라인 흐름

```
1. GitHub Actions cron 트리거 (매일 00시, 12시)
        │
2. 구독 중인 영화 목록 Firestore에서 조회
        │
3. 영화별 스크래핑 실행 (CGV / 롯데 / 메가박스 병렬)
        │
4. 전날 스냅샷과 오늘 수집 결과 diff 비교
   (추가 / 삭제 / 수정 감지)
        │
    ┌───┴───┐
 변경 있음   변경 없음
    │           │
5. Firestore   종료
   스냅샷 갱신
    │
6. 해당 영화 구독 유저 조회
    │
7. Telegram Bot으로 변경 내역 메시지 발송
```

## 8. 리스크 및 대응 방안

| 리스크                                    | 대응 방안                                       |
| ----------------------------------------- | ----------------------------------------------- |
| 영화관 사이트 구조 변경으로 스크래핑 중단 | 사이트별 파서 모듈 분리, 에러 알림 별도 발송    |
| GitHub Actions 무료 한도 초과             | 00시 1회 실행으로 한도 부담 거의 없음           |
| Telegram Bot API 제한                     | 메시지 큐 처리, 일괄 발송 지양                  |
| Firestore 읽기/쓰기 비용                  | 캐싱 레이어 적용, 불필요한 리얼타임 리스너 제거 |
| 스크래핑 차단 (IP 밴)                     | GitHub Actions IP 풀 활용, User-Agent 로테이션  |
