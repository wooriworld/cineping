> **프로젝트명: cineping  
> \*\*버전:\*\* v1.4
> **작성일:\*\* 2026.03.13

## 1. 프로젝트 개요

### 1.1 서비스 소개

영화 상영 스케줄을 자동 수집하여 변경사항이 생겼을 때 알림을 발송하는 서비스. 웹에서는 통합 스케줄을 한눈에 조회할 수 있다.

### 1.2 핵심 문제 정의

국내 멀티플렉스(CGV·롯데시네마·메가박스)는 상영 스케줄 업데이트 시점을 별도로 공지하지 않는다. 사용자는 원하는 영화의 스케줄이 언제 열리는지 알 수 없어, 예매 기회를 놓치지 않으려면 각 사이트를 반복적으로 직접 방문해 확인하는 수밖에 없다.

이는 **정보 비대칭으로 인한 사용자 경험 저하**이며, 특히 인기 영화일수록 스케줄 오픈 직후 좌석이 빠르게 소진되기 때문에 확인 타이밍이 예매 성공 여부를 직접적으로 좌우한다.

본 서비스는 이 반복적인 수동 확인 행위를 자동화하여, **스케줄 변경이 발생한 시점에 사용자에게 즉시 알림을 전달**함으로써 예매 골든타임을 놓치지 않도록 한다.

### 1.3 목표 사용자

- 개봉 예정 영화의 스케줄이 언제 풀리는지 기다리는 영화 팬
- 매일 영화사이트를 직접 들어가 확인하는 수고를 줄이고 싶은 관객

## 2. 주요 기능 정의

### 2.1 스케줄 수집 배치 (Railway Cron)

| 항목      | 내용                                                 |
| --------- | ---------------------------------------------------- |
| 수집 주기 | 매일 9시, 18시 2회 (Railway Cron 스케줄)             |
| 수집 대상 | 네이버 현재 상영 영화 / KOFA 시네마테크 / 에무시네마 |
| 저장소    | Supabase (PostgreSQL)                                |

**소스별 수집 방식**

| 소스       | 방식                                                           |
| ---------- | -------------------------------------------------------------- |
| 네이버     | 스케줄 API HTTP 직접 호출 + cheerio 파싱                       |
| KOFA       | 시네마테크 스케줄 페이지 HTTP 호출 + cheerio HTML 파싱         |
| 에무시네마 | 상영시간표 이미지 다운로드 → Gemini Vision API OCR → JSON 추출 |

**수집 데이터 항목**

영화 정보

- 영화 제목 (한글), 영어 제목
- 소스별 고유 ID, 수집 출처
- 포스터 이미지 URL
- 개봉일
- 등록 시각 (KST)

스케줄 정보

- 영화관 체인, 지점명
- 상영 날짜, 시작 시간, 종료 시간
- 상영관 타입 (IMAX, 4DX, 일반 등)
- 영어 자막 여부
- 예매 링크
- 마지막 수집 시각 (KST)

### 2.2 알림 시스템 (텔레그램)

**알림 트리거 조건 (스크래핑 후 직전 스냅샷과 diff 비교)**

- 새로운 영화가 추가된 경우
- 기존 영화에 새로운 상영 회차가 추가된 경우 (단, 당일 신규 등록 영화는 제외)

**알림 메시지 형식**

```
🔥🔥 Movie Update Alert

New Movies (N)
🎬 [ 영화제목1 ]
🎬 [ 영화제목2 ]
🎬 [ 영화제목3 ]
... and N more

Showtime Updates (N)
🎬 [ 영화제목1 ]
🎬 [ 영화제목2 ]
... and N more

🔗 View Details
{서비스 URL}?t={token}
```

> - 신규 영화만 있거나 스케줄 업데이트만 있는 경우 해당 섹션만 포함
> - 바로가기 URL에는 8자리 랜덤 토큰(`url_tokens` 테이블 저장)을 사용해 관련 영화 목록을 필터링하여 표시

### 2.3 웹 서비스 (Vue.js)

#### 페이지 구성

| 페이지      | 경로     | 설명                                                 |
| ----------- | -------- | ---------------------------------------------------- |
| 영화 목록   | `/`      | 현재 상영 영화 조회 / 검색 / 배지 필터 / 스케줄 팝업 |
| 어드민      | `/admin` | 영화 수집, 전체·개별 스케줄 수집, 영화 삭제          |
| 사용자 관리 | `/users` | 텔레그램 연동 및 알림 조건 설정                      |

#### 주요 UI 컴포넌트

**① 영화 목록 (MoviesPage)**

- 현재 상영 중인 영화를 그리드 카드로 표시
- 제목·영문명 통합 검색
- NEW / UPDATE / ENG 배지 필터
- 영화 카드 클릭 시 스케줄 상세 팝업

**② 날짜 선택 컴포넌트**

- 오늘부터 7일치 날짜 탭 가로 나열
- 해당 날짜에 스케줄 없으면 비활성(disabled) 처리
- 신규 스케줄이 있는 날짜에 NEW 배지 표시

**③ 필터 컴포넌트**

- 지역 탭: 한국의 각 지역 목록 드롭다운 형태로 중복 선택 가능하게
- 극장 탭: 한국의 영화관 목록 드롭다운 형태로 "전체"가 디폴트
- 상영관 탭: 일반관 / 특별관
- 정렬 기준: 극장순 / 시간순

**④ 스케줄 컴포넌트**

- 극장별 카드 단위로 시간표 표시
- 시간 버튼 가로 스크롤 (스크린타입 함께 표시)
- 시간 버튼 클릭 시 해당 영화관 예매 페이지로 이동
- 신규 스케줄에 NEW 배지, 영어 자막 스케줄에 ENG 배지 표시

**사용자 관리 페이지 (/users)**

- 사용자 추가 / 수정 / 삭제
- 텔레그램 Chat ID 입력 및 연동
- 감시 영화(watchlist) 관리
- 알림 조건 설정 (새 스케줄 추가 / 삭제 / 변경)
- 선호 극장 체인 다중 선택

## 5. 데이터 모델 (Supabase)

### `movies` 테이블

```json
{
  "id": "uuid",
  "title": "왕과 사는 남자",
  "englishTitle": "The King's Man",
  "sourceId": "244329",
  "source": "NAVER",
  "poster": "https://...",
  "releaseDate": "2026-02-04",
  "createdAt": "2026-03-14T00:00:00+09:00"
}
```

> 필드 설명
>
> - source: 수집 출처 (`NAVER` / `KOFA` / `EMUCINE`)
> - sourceId: 소스별 고유 ID (NAVER: 네이버 영화 ID, KOFA: KOFA 영화 ID, EMUCINE: 임의 6자리)
> - releaseDate: 개봉일 (YYYY-MM-DD) — 정렬 및 NEW 뱃지 표시용
> - createdAt: KST ISO 8601 — 당일 신규 영화 여부 판별용

---

### `schedules` 테이블

```json
{
  "id": "uuid",
  "movieId": "uuid",
  "chain": "메가박스",
  "theater": "메가박스 강동",
  "date": "2026-03-14",
  "startTime": "16:00",
  "endTime": "18:07",
  "screenType": "2D",
  "bookingUrl": "https://...",
  "hasEnglishSubtitle": false,
  "lastUpdatedAt": "2026-03-14T12:00:00+09:00"
}
```

> 필드 설명
>
> - chain: 영화관 체인 (`CGV` / `롯데시네마` / `메가박스` / `씨네Q` / `KOFA` / `EMUCINE`)
> - hasEnglishSubtitle: 영어 자막 상영 여부 — ENG 배지 표시 기준
> - unique key: `date + theater + startTime` — diff 비교 및 중복 방지용
> - lastUpdatedAt: KST ISO 8601 — NEW 배지 표시 기준

---

### `url_tokens` 테이블

```json
{
  "id": "uuid",
  "token": "aB3xYz9K",
  "ids": ["244329", "198432"],
  "created_at": "2026-03-26T09:00:00+09:00"
}
```

> 필드 설명
>
> - token: 8자리 랜덤 문자열 — Telegram 바로가기 URL 쿼리 파라미터(`?t=`)로 사용
> - ids: 해당 알림에 포함된 sourceId 배열 — 웹앱에서 해당 영화만 필터링하여 표시

## 6. 배치 파이프라인 흐름

```
1. Railway Cron 트리거 (KST 매일 09:00 / 18:00)
        │
        ▼ run-scrape.js
2. 네이버 영화 API
   → 신규 영화 INSERT / 영어 제목 누락 시 UPDATE
        │
3. Supabase에서 전체 영화 목록 조회
        │
4. 영화별 네이버 스케줄 API 순차 호출 (딜레이 1.5초)
   → diff 비교 (date + theater + startTime 기준)
   → 신규: INSERT / 사라진 것: DELETE
        │
5. KOFA 시네마테크 HTML 파싱
   → 영어자막 영화·스케줄 수집 → diff 후 저장
   → hasEnglishSubtitle = true 플래그
        │
6. 에무시네마 상영시간표 이미지 다운로드
   → Gemini Vision API OCR → JSON 추출
   → Eng 자막 스케줄만 필터링 → diff 후 저장
   → hasEnglishSubtitle = true 플래그
   (실패 시 해당 수집만 건너뜀, 전체 배치 중단 없음)
        │
    ┌───┴───┐
 변경 있음   변경 없음
    │
7. url_tokens 테이블에 토큰 생성 (sourceId 배열 저장)
        │
8. Telegram 통합 알림 발송
```

## 8. 리스크 및 대응 방안

| 리스크                             | 대응 방안                                                                                                                  |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| 네이버 API 스펙 변경으로 수집 중단 | 파서 모듈 분리 (`naverMovieApiParser`, `naverScheduleApiParser`), 에러 발생 시 해당 파서만 실패 처리 후 텔레그램 알림 발송 |
| KOFA / 에무시네마 파서 오류        | 소스별 독립 try/catch 처리, 실패 시 해당 수집만 건너뛰고 나머지 배치 정상 진행                                             |
| Gemini Vision API 장애·할당량 초과 | 에무시네마 수집 실패 시 개별 처리 후 계속 진행, 전체 배치 영향 없음. 무료 할당량 모니터링 필요                             |
| 에무시네마 이미지 URL 패턴 변경    | 파일명 고정 참조 대신 `img[src^="/about/emuartspace/all/"]` 패턴으로 동적 탐색하여 파일명 변경에 무관                      |
| Railway 무료 티어 한도 초과        | 하루 2회 배치 실행(`run-scrape.js`), 실행 시간·메모리 사용량 모니터링 필요                                                 |
| Telegram Bot API 속도 제한         | 영화별 스케줄 수집 간 1.5초 딜레이 적용, 알림은 수집 완료 후 1건으로 통합 발송                                             |
| Supabase 무료 티어 용량/요청 한도  | 스케줄 diff 방식으로 불필요한 INSERT 최소화, 사라진 스케줄 DELETE로 데이터 적재 방지                                       |
| 스크래핑 차단 (IP 밴)              | 네이버 공개 API 직접 호출 방식으로 HTML 파싱 의존도 없음, Railway 고정 IP 풀 활용                                          |
| GitHub Actions 무료 한도 초과      | 프론트엔드 배포 전용으로만 사용 (push 시 1회, 1~2분 소요), 월 사용량 미미                                                  |

- **카카오 알림톡 / Discord Webhook** 연동 (알림 채널 확장)

# 🎬 cineping 수익화 모델 제안서 (Monetization Strategy)

본 제안서는 **실시간 영화 상영 스케줄 수집 및 알림**이라는 시네핑의 핵심 가치를 기반으로, 서비스의 지속 가능성과 수익성을 확보하기 위한 단계별 전략을 담고 있습니다.

---

## 1. B2C: 프리미엄 개인화 서비스 (Subscription)

사용자의 '예매 성공 확률'을 극대화하는 기능을 유료 멤버십 형태로 제공합니다.

- **초고속 스캔권 (High-Frequency Scrape):** \* 현재 1일 2회(09시, 18시)인 수집 주기를 프리미엄 유저에 한해 1시간 단위 또는 실시간급으로 단축하여 제공.
- **개인 맞춤형 타겟 알림:** \* **무료:** 전체 공지방 알림 수신.
  - **유료:** 특정 영화 + 특정 영화관(예: 용산 아이맥스, 에무시네마) 조합을 설정하여 해당 조건 충족 시에만 1:1 즉시 알림 수신.
- **감시 슬롯 확장:** \* 관심 영화 등록 개수 제한 해제 (예: 무료 2개 → 유료 무제한).

## 2. B2B: 독립·예술영화관 파트너십 (Native Ad)

대형 멀티플렉스에 비해 홍보 채널이 부족한 독립예술영화관과의 상생 모델입니다.

- **상단 노출 및 하이라이트:** \* 웹사이트 스케줄 조회 시, 제휴 영화관의 기획전이나 특별 회차를 상단에 고정하거나 시각적으로 강조.
- **큐레이션 알림 하단 광고:** \* 신규 스케줄 업데이트 알림 하단에 제휴 영화관의 관련 영화 추천 정보를 네이티브 광고 형태로 삽입.
- **예매 연결 수수료:** \* 시네핑의 '예매 링크'를 통해 각 영화관 공식 예매 사이트로 유입되는 트래픽에 대한 성과 보수 협의.

## 3. 타겟 마케팅 및 제휴 (Affiliate & Global)

시네핑만의 차별화된 데이터(영어 자막, 굿즈 정보 등)를 활용합니다.

- **영어 자막(ENG) 특화 서비스:** \* 기획서의 핵심 차별점인 '영어 자막 필터링' 데이터를 활용, 국내 거주 외국인 커뮤니티나 관련 서비스와 제휴하여 데이터 API 유료화 또는 전용 광고 유치.
- **영화 굿즈 커머스 연계:** \* 스페셜 티켓(OT), 포스터 등 굿즈 수집가를 위해 해당 영화의 굿즈 배포 정보와 함께 관련 보관함 판매 페이지 링크(제휴 마케팅) 제공.
- **할인 예매권 유통:** \* 오픈마켓(기프티콘, 예매권 할인 판매처)과의 제휴를 통해 스케줄 하단에 최저가 예매권 구매 링크 배치.

## 4. 데이터 비즈니스 (Data-as-a-Service)

수집된 실시간 상영 스케줄 데이터를 가공하여 B2B로 활용합니다.

- **상영 스케줄 분석 리포트:** \* 중소 배급사나 영화 홍보사를 대상으로 자사/경쟁사 영화의 상영 시간대 점유율, 지역별 편성 현황 등을 시각화한 대시보드 또는 리포트 판매.
- **예매 골든타임 데이터:** \* 각 영화관별 스케줄 오픈 패턴을 분석하여 마케팅 적기 및 예매 집중 시간대 데이터 제공.
