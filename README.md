> **프로젝트명: cineping  
> \*\*버전:\*\* v1.3
> **작성일:\*\* 2026.03.13

## 1. 프로젝트 개요

### 1.1 서비스 소개

영화 상영 스케줄을 자동 수집하여 변경사항이 생겼을 때 알림을 발송하는 서비스. 웹에서는 통합 스케줄을 한눈에 조회할 수 있다.

### 1.2 핵심 문제 정의

국내 멀티플렉스(CGV·롯데시네마·메가박스)는 상영 스케줄 업데이트 시점을 별도로 공지하지 않는다. 사용자는 원하는 영화의 스케줄이 언제 열리는지 알 수 없어, 예매 기회를 놓치지 않으려면 각 사이트를 반복적으로 직접 방문해 확인하는 수밖에 없다.

이는 **정보 비대칭으로 인한 사용자 경험 저하**이며, 특히 인기 영화일수록 스케줄 오픈 직후 좌석이 빠르게 소진되기 때문에 확인 타이밍이 예매 성공 여부를 직접적으로 좌우한다.

본 서비스는 이 반복적인 수동 확인 행위를 자동화하여, **스케줄 변경이 발생한 시점에 사용자에게 즉시 알림을 전달**함으로써 예매 골든타임을 놓치지 않도록 한다.

### 1.3 목표 사용자

- 개봉 예정 영화의 스케줄이 언제 풀리는지 기다리는 영화 팬
- 매일 영화사이트를 직접 들어가 확인하는 수고를 줄이고 싶은 관객

## 2. 주요 기능 정의

### 2.1 스케줄 수집 배치 (GitHub Actions)

| 항목      | 내용                                            |
| --------- | ----------------------------------------------- |
| 수집 주기 | 매일 00시, 12시 2회 (GitHub Actions cron)       |
| 수집 대상 | 네이버 "현재 상영 영화" 통합 수집               |
| 수집 방식 | 네이버 스케줄 API HTTP 직접 호출 + cheerio 파싱 |
| 저장소    | Supabase (PostgreSQL)                           |

**수집 데이터 항목**

영화 정보

- 영화 제목 (한글), 영어 제목
- 네이버 영화 ID, 포스터 이미지 URL
- 개봉일 (YYYY.MM.DD)
- 등록 시각 (KST)

스케줄 정보

- 영화관 체인, 지점명
- 상영 날짜, 시작 시간, 종료 시간
- 상영관 타입 (IMAX, 4DX, 일반 등)
- 예매 링크
- 마지막 수집 시각 (KST)

### 2.2 알림 시스템 (텔레그램)

**알림 트리거 조건 (스크래핑 후 직전 스냅샷과 diff 비교)**

- 새로운 영화가 추가된 경우 → 신규 영화 알림 발송
- 기존 영화에 새로운 상영 회차가 추가된 경우 → 스케줄 업데이트 알림 발송

**알림 메시지 형식**

신규 영화 알림

```
🔥🔥 신규 영화 업데이트 N건

🎬 [ 영화제목1 ]
🎬 [ 영화제목2 ]
🎬 [ 영화제목3 ]
... 외 N개

🔗 바로가기:
{서비스 URL}
```

스케줄 업데이트 알림

```
🔥🔥 영화 스케줄 업데이트 N건

🎬 [ 영화제목1 ]
🎬 [ 영화제목2 ]
🎬 [ 영화제목3 ]
... 외 N개

🔗 바로가기
{서비스 URL}
```

### 2.3 웹 서비스 (Vue.js)

#### 페이지 구성

| 페이지      | 경로     | 설명                                                 |
| ----------- | -------- | ---------------------------------------------------- |
| 영화 목록   | `/`      | 현재 상영 영화 조회 / 검색 / 배지 필터 / 스케줄 팝업 |
| 어드민      | `/admin` | 영화 수집, 전체·개별 스케줄 수집, 영화 삭제          |
| 사용자 관리 | `/users` | 텔레그램 연동 및 알림 조건 설정                      |

#### 주요 UI 컴포넌트

**① 영화 목록 (MoviesPage)**

- 현재 상영 중인 영화를 그리드 카드로 표시
- 제목·영문명 통합 검색
- NEW / UPDATE 배지 필터 드롭다운
- 영화 카드 클릭 시 스케줄 상세 팝업

**② 날짜 선택 컴포넌트**

- 오늘부터 7일치 날짜 탭 가로 나열
- 해당 날짜에 스케줄 없으면 비활성(disabled) 처리
- 신규 스케줄이 있는 날짜에 NEW 배지 표시

**③ 필터 컴포넌트**

- 지역 탭: 한국의 각 지역 목록 드롭다운 형태로 중복 선택 가능하게
- 극장 탭: 한국의 영화관 목록 드롭다운 형태로 "전체"가 디폴트
- 정렬 기준: 극장순 / 시간순

**④ 스케줄 컴포넌트**

- 극장별 카드 단위로 시간표 표시
- 시간 버튼 가로 스크롤 (스크린타입 함께 표시)
- 시간 버튼 클릭 시 해당 영화관 예매 페이지로 이동
- 신규 스케줄에 NEW 배지 표시

**어드민 페이지 **

- 현재 상영 영화 수집 (네이버 API)
- 전체 스케줄 수집 / 영화별 개별 스케줄 수집
- 영화 목록 테이블: 개봉일 desc → 등록일 desc → 스케줄 수 desc 정렬
- 개봉일 기준 NEW 배지, 스케줄 수 표시
- 영화 삭제, 영화별 스케줄 전체 삭제

**사용자 관리 페이지 (/users)**

- 사용자 추가 / 수정 / 삭제
- 텔레그램 Chat ID 입력 및 연동
- 감시 영화(watchlist) 관리
- 알림 조건 설정 (새 스케줄 추가 / 삭제 / 변경)
- 선호 극장 체인 다중 선택

## 5. 데이터 모델 (Supabase)

### `movies` 테이블

```json
{
  "id": "uuid",
  "title": "왕과 사는 남자",
  "englishTitle": "The King's Man",
  "naverMovieId": "244329",
  "poster": "https://...",
  "releaseDate": "2026.02.04",
  "createdAt": "2026-03-14T00:00:00+09:00"
}
```

> 필드 설명
>
> - naverMovieId: 네이버 영화 고유 ID — 스케줄 API 호출 시 사용
> - releaseDate: 개봉일 (점 구분, YYYY.MM.DD) — 정렬 및 NEW 뱃지 표시용
> - createdAt: KST ISO 8601 — 당일 신규 영화 여부 판별용

---

### `schedules` 테이블

```json
{
  "id": "uuid",
  "movieId": "uuid",
  "chain": "메가박스",
  "theater": "메가박스 강동",
  "date": "2026-03-14",
  "startTime": "16:00",
  "endTime": "18:07",
  "screenType": "2D",
  "bookingUrl": "https://...",
  "lastUpdatedAt": "2026-03-14T12:00:00+09:00"
}
```

> 필드 설명
>
> - chain: 영화관 체인 ("CGV" / "롯데시네마" / "메가박스" / "씨네Q")
> - unique key: `date + theater + startTime` — diff 비교 및 중복 방지용
> - lastUpdatedAt: KST ISO 8601 — NEW 배지 표시 기준

---

### `users` 테이블

```json
{
  "uid": "user_uid",
  "telegramChatId": "123456789",
  "watchlist": ["movie_uuid_1", "movie_uuid_2"],
  "alertConditions": {
    "newSchedule": true,
    "deletedSchedule": false,
    "modifiedSchedule": true,
    "preferredChains": ["CGV", "메가박스"]
  }
}
```

> 필드 설명
>
> - watchlist: 감시 중인 영화 ID 배열
> - alertConditions: 알림 트리거 조건 (신규 추가 / 삭제 / 수정)
> - preferredChains: 알림 받을 극장 체인 목록 (빈 배열이면 전체)

## 6. 배치 파이프라인 흐름

```
1. GitHub Actions cron 트리거 (KST 매일 09:00 / 18:00)
        │
2. 네이버 영화 API로 현재 상영 영화 목록 수집
        │
3. Supabase movies 테이블과 비교
   → 신규 영화 INSERT / 영어 제목 누락 시 UPDATE
        │
4. Supabase에서 전체 영화 목록 조회
        │
5. 영화별 네이버 스케줄 API 순차 호출 (딜레이 1.5초)
        │
6. 기존 schedules와 diff 비교 (date + theater + startTime 기준)
   → 신규: INSERT / 변경: UPDATE / 사라진 것: DELETE
        │
    ┌───┴───┐
 변경 있음   변경 없음
    │           │
7. Telegram    종료
   통합 알림 발송
   (신규 영화 + 스케줄 업데이트 묶어서 1건)
   바로가기 URL: /?id=naverMovieId1,naverMovieId2,...
```

## 8. 리스크 및 대응 방안

| 리스크                              | 대응 방안                                                              |
| ----------------------------------- | ---------------------------------------------------------------------- |
| 네이버 API 스펙 변경으로 수집 중단  | 파서 모듈 분리 (`naverMovieApiParser`, `naverScheduleApiParser`), 에러 발생 시 해당 파서만 실패 처리 후 텔레그램 알림 발송 |
| GitHub Actions 무료 한도 초과       | 하루 2회 실행, 월 약 60회 × 5분 ≈ 300분 소비 (무료 한도 2,000분 내 여유) |
| Telegram Bot API 속도 제한          | 영화별 스케줄 수집 간 1.5초 딜레이 적용, 알림은 수집 완료 후 1건으로 통합 발송 |
| Supabase 무료 티어 용량/요청 한도   | 스케줄 diff 방식으로 불필요한 INSERT 최소화, 사라진 스케줄 DELETE로 데이터 적재 방지 |
| 스크래핑 차단 (IP 밴)               | 네이버 공개 API 직접 호출 방식으로 HTML 파싱 의존도 없음, GitHub Actions IP 풀 활용 |

## 9. 향후 확장 기능 (v2)

- **사용자별 watchlist 알림** — 현재는 채널 전체 알림만 발송, 감시 영화 지정 후 개인별 텔레그램 알림
- **특정 지역/지점 필터 알림** — 예: "강남 CGV만 알림" 조건 설정
- **수집 주기 조정** — 현재 하루 2회 고정, 이벤트 기반(영화 개봉일 전후 집중 수집) 전환
- **PWA 푸시 알림** 지원
- **카카오 알림톡 / Discord Webhook** 연동 (알림 채널 확장)
